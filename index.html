BiZOOPeに接続ができなかった際の対処法

  1. 履歴、Cookieなど、または証明書マネージャーよりキャッシュのクリアをお願いします。

    Edgeの場合

    1. ブラウザ右上にある[・・・]を選択
    2. [設定]を選択
    3. 左メニューより [プライバシー、検索、サービス]を選択
    4. 「閲覧データをクリア」より行えます

    Chromeの場合

    1. ブラウザ右上にある[・・・]を選択
    2. [設定]を選択
    3. 左メニューより[プライバシーとセキュリティ]を選択
    4. 「閲覧履歴データの削除」より行えます

    履歴、Cookieなどを削除したくない場合、シークレットモードでお試しください

    Edgeの場合

    1. ブラウザ右上にある[・・・]を選択
    2. [設定]を選択
    3. [新しいInPrivateウィンドウ]を選択

    Chromeの場合

    1. ブラウザ右上にある[・・・]を選択
    2. [設定]を選択
    3. [新しいシークレットウィンドウ]を選択
  2. クライアント証明書は正常にインストールできているか確認をお願いします。

    3. Edgeの場合

    1. ブラウザ右上にある[・・・]を選択
    2. [設定]を選択
    3. 左メニューより [プライバシー、検索、サービス]を選択
    4. セキュリティより証明書の管理をクリック
    5. 「個人」のタブに発行者が「okuwa.bizoo.cloud CA」の証明書があれば、クライアント証明書は正常にインストールされています。

    Chromeの場合

    1. ブラウザ右上にある[・・・]を選択
    2. [設定]を選択
    3. 左メニューより[プライバシーとセキュリティ]を選択
    4. 「セキュリティ」→[証明書の管理]をクリック
    5. 「個人」のタブに発行者が「okuwa.bizoo.cloud CA」の証明書があれば、クライアント証明書は正常にインストールされています。

    クライアント証明書がインストールされていた場合

      御社のネットワーク管理者様にプロキシサーバーの利用状況をご確認頂ください。

      もしプロキシサーバーをご利用されているようでしたら、御社のプロキシサーバーでBiZOOPeのURLに対して、クライアント証明書を使っての接続ができないような設定がされていないか、ご確認お願い致します。

    クライアント証明書がインストールされていない場合

      前担当者様がお持ちの場合がありますので、ご確認お願いします。

      「okuwa-info@bizoo.cloud」からクライアント証明書が送られている可能性がありますので、過去メールからご確認お願いします。

      上記でクライアント証明書が見つからなかった場合、再送を行う必要がありますので、ご契約企業様までご連絡をお願いします。
